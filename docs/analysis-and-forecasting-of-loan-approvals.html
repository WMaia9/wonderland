Project 6 Analysis and Forecasting of Loan Approvals

6.1 Introduction

This paper analyzes data on bank loans

6.1.1 Objectives of the work

In this work, the objective is to analyze data on lending using data science tools. The main question we will seek to answer is whether we can predict the granting of loans using customer history.

6.1.2 Analysis

This work uses descriptive statistics and data visualization to find key figures in understanding the distribution, count and relationship between variables. Since the objective of the work is to make lending predictions for users, classification algorithms of the supervised learning family of machine learning models are implemented.

6.1.3 Evaluation

The work ends with an evaluation of the selected machine learning model with a set of validation data. The output of the forecasts can be verified using a confusion matrix and Score metrics from the Cross validation.

6.2 Data

The project has two sets of data provided by the MBA program of the University of São Paulo in Data Science, they are, “File Treino.csv” that will be used for all data analysis, and the second, “File Teste.csv”, which will be used to validate the algorithms.

6.2.1 Data Characteristics

The Training File has 614 rows and 13 columns, this is not a good sign, since we have very little data for machine learning.

Columns in the dataset include:

  • Loan_ID: unique loan id
  • Gender: Male / Female
  • MarriedMarried:applicant (Y / N)
  • Dependents: User dependent numbers
  • EducationEducation:level
  • Sel_Empoyedemployed: Self-(Y / N)
  • ApplicantIncome: Income from applicant
  • Coapplicantlncome: Revenue from the co-applicant
  • LoanAmount: Loan amount in thousands
  • Loan_Amount_Term: Loan term in months
  • Credit_History: Credit history meets the guidelines
  • Property_Area: Urban / Semi-Urban / Rural
  • Loan_Status: Loan approved (Y / N)

6.5.2 Logistic Regression

We will start with the logistic regression algorithm. One way would be to include all variables in the model, however this can result in overfitting or over-adjustment.

We can make some intuitive hypotheses to start our model. We can, for example, assume that the chance of getting a loan will be greater for:
  1. Customers who have a credit history (remember that we observed this in the exploration of the data);
  2. Customers and guarantors with higher yields (it seems reasonable for the institution to consider that there is less risk in lending so that it has higher yields);
  3. Customers with a higher level of education (it seems reasonable for the institution to consider that customers with a higher level of education better assess their ability to repay the loan);
  4. Properties in urban areas with high growth prospects (it seems reasonable to imagine that properties in urban areas with high growth expectations will increase more and reduce the institution’s risk if the client does not pay the loan).

We will start by using only the variable “Credit History” (Credit_History) as a predictor variable and loan status (Loan_Status) as a dependent variable.

6.6 Conclusion

Data analysis is a fundamental area for Data Scientist, with which we were able to make several hypotheses and draw conclusions to be used in our machine learning model. Knowing a little about the business model was essential to deal with outlier and extreme houses. 

Another important aspect was the use of resource engineering, which was a simple implementation, but nevertheless it enabled us to improve the accuracy of our Machine Learning model, proving to be an essential tool if we want to take our precision model seriously.  

We saw that more sophisticated models do not always bring us better results, it is useless to use models that consume more computational power without understanding them or if they give you the same result as simpler models. In order to take better advantage of the more sophisticated models, it was necessary to use resource engineering and different hyperparameters in the model to avoid overfitting. In the method of sets we did not have an improvement in the model as expected, this is perhaps one to analyze with different variables and parameters in the model. 

Finally, Data Science is multidisciplinary and goes far beyond running ready-made algorithms and statistical techniques, it is necessary to do more research on the business model being analyzed and to question very obvious results.
